QUE ES CSS

  • CSS: Cascading Style Sheets. hojas de estilo en cascada.
  • Es el lenguaje que define el estilo de los elementos de un documento HTML.
  • Es el lenguaje para controlar el diseño y apariencia de las webs hechas en HTML.
  • En 1998 se publica CSS NIVEL 2.
  • La Versión (2009) 2.1 es la que utilizan hoy en día todos los navegadores
  • Por el momento ningún navegador tiene soporte completo de CSS 2.1

VENTAJAS DE USAR CSS

1. SEPARACIÓN DEL CONTENIDO Y PRESENTACIÓN

  • Las hojas de estilo generalmente se encuentran en archivos separados del código principal.
  • Esto permite que un equipo de trabajo pueda hacer tareas del código principal independientemente del diseño.

2. FLEXIBILIDAD

  • Se puede modificar el archivo de estilo y diseño sin cambiar el contenido.

UNIFICACIÓN DEL DISEÑO DE LAS PAGINAS DEL SITIO

  • Permite mantener una misma apariencia en cada una de las páginas y modificarlas en un mismo proceso.
  • Ya que todas las paginas están enlazadas a la hoja de estilo, agilizando el proceso y minimizando el trabajo.

4. OPTIMIZACIÓN DE LOS TIEMPOS DE CARGA Y DE TRAFICO EN EL SERVIDOR

  • Al haber dividido contenido se obtienen archivos más ligeros, reportando:
    1. reducir notablemente el tiempo de carga del sitio en el navegador.
    2. reducir el volumen de tráfico del servidor

5. PRECISIÓN O ELASTICIDAD

  • el tamaño y posicionamiento de los elementos podrá ser exacto.
  • se puede indicar al navegador en que píxel debe colocar un diseño.
  • se puede usar medidas variables o relativas que permitan expandir o reducir el contenido de la ventana de navegación.

6. ACCESIBILIDAD Y ESTRUCTURACIÓN

  • la combinación de CSS y marcadores descriptivos permite que la página se vea corectamente con o sin hoja de estilos, ya que la información se mantendrá estructurada y ordenada.
  • será accesible sin ningún tipo de problemas tanto por navegadores antiguos o sin soporte CSS, como para personas con algún tipo de discapacidad.

7. LIMPIEZA DEL CÓDIGO FUENTE

  • si escribimos una hoja de estilo independiente, el código fuente va a resultar más agil las tareas de localización de las líneas que busquemos.

8. COMPATIBILIDAD Y CONTINUIDAD

  • las reglas establecidad en CSS-1 se mantienen en CSS-2.
  • Por lo que la compatibilidad de las especificaciones CSS anteriores estará siempre garantizada.

9. ESTANDARIZACIÓN

  • la adopción de estándares por la W3C ofrece la ventaja de la compatibilidad del código entre los diferentes navegadores web.
  • el uso del estándar CSS de la W3C evita visualizaciones incorectas de las web en los distintos navegadores.

10. PERMITE LA DIFERENCIACIÓN DE ESTILOS PARA IMPRIMIR (VISUALIZAR EN PANTALLA)

  • CSS permite maquetar separadamente el contenido de la web para mostrarlo en la pantalla.
