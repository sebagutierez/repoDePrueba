  • CSS: Cascading Style Sheets. hojas de estilo en cascada.
  • Es el lenguaje que define el estilo de los elementos de un documento HTML.
  • Es el lenguaje para controlar el diseño y apariencia de las webs hechas en HTML.
  • En 1998 se publica CSS NIVEL 2.
  • La Versión (2009) 2.1 es la que utilizan hoy en día todos los navegadores
  • Por el momento ningún navegador tiene soporte completo de CSS 2.1

VENTAJAS DE USAR CSS

1. SEPARACIÓN DEL CONTENIDO Y PRESENTACIÓN

  • Las hojas de estilo generalmente se encuentran en archivos separados del código principal.
  • Esto permite que un equipo de trabajo pueda hacer tareas del código principal independientemente del diseño.

2. FLEXIBILIDAD

  • Se puede modificar el archivo de estilo y diseño sin cambiar el contenido.

UNIFICACIÓN DEL DISEÑO DE LAS PAGINAS DEL SITIO

  • Permite mantener una misma apariencia en cada una de las páginas y modificarlas en un mismo proceso.
  • Ya que todas las paginas están enlazadas a la hoja de estilo, agilizando el proceso y minimizando el trabajo.

4. OPTIMIZACIÓN DE LOS TIEMPOS DE CARGA Y DE TRAFICO EN EL SERVIDOR

  • Al haber dividido contenido se obtienen archivos más ligeros, reportando:
    1. reducir notablemente el tiempo de carga del sitio en el navegador.
    2. reducir el volumen de tráfico del servidor

5. PRECISIÓN O ELASTICIDAD

  • el tamaño y posicionamiento de los elementos podrá ser exacto.
  • se puede indicar al navegador en que píxel debe colocar un diseño.
  • se puede usar medidas variables o relativas que permitan expandir o reducir el contenido de la ventana dennavegación.

6. ACCESIBILIDAD Y ESTRUCTURACIÓN

  • la combinación de CSS y marcadores descriptivos permite que la página se vea corectamente con o sin hoja de estilos, ya que la información se mantendrá estructurada y ordenada.
  • será accesible sin ningún tipo de problemas tanto por navegadores antiguos o sin soporte CSS, como para personas con algún tipo de discapacidad.

7. LIMPIEZA DEL CÓDIGO FUENTE

  • si escribimos una hoja de estilo independiente, el código fuente va a resultar más agil las tareas de localización de las líneas que busquemos.

8. COMPATIBILIDAD Y CONTINUIDAD

  • las reglas establecidad en CSS-1 se mantienen en CSS-2.
  • Por lo que la compatibilidad de las especificaciones CSS anteriores estará siempre garantizada.

9. ESTANDARIZACIÓN

  • la adopción de estándares por la W3C ofrece la ventaja de la compatibilidad del código entre los diferentes navegadores web.
  • el uso del estándar CSS de la W3C evita visualizaciones incorectas de las web en los distintos navegadores.

10. PERMITE LA DIFERENCIACIÓN DE ESTILOS PARA IMPRIMIR (VISUALIZAR EN PANTALLA)

  • CSS permite maquetar separadamente el contenido de la web para mostrarlo en la pantalla.

QUE ES UNA REGLA CSS

  • Es una forma adecuada de escribir una instrucción mediante código CSS
  • La manera en que se debe escribir código CSS para que sea interpretado en un navegador.
  • Formada por dos partes:
    1. un sector
    2. una declaración: donde se encuentran las propiedades con su respectivo valor.
  • Partes de una regla CSS
    1. EL SELECTOR: quien indica a que elemento HTML se aplica el código CSS.
    2. LA PROPIEDAD: la característica que se desea definir. Ejemplo: color.
    3. EL VALOR: es el valor que tomará la propiedad. Ejemplo: green.
    4. PUNTO Y COMA: una declaración siempre debe terminar con ;
    5. EJEMPLO DE UNA REGLA CSS: h1 {color: red;}
    6. (h1): el selector h1 apunta a un elemento de HTML.
    7. {color: red;}: la declaración siempre va dentro de llaves y contiene a la propiedad y su valor cerrada con un ;
    8. (color): se define la pripiedad de color
    9. (red): el valor del color es red.

SELECTORES CSS

SELECTOR DE TIPO ETIQUETA

selecciona todos los elementos que coinciden con el nombre del elemento especificado.

Ejemplo: h1{propiedad: valor;}

Ejemplo: (p class=""etiqueta)

SELECTOR DE CLASE

selecciona todos los elementos que tiene el atributo de class especificado.

Ejemplo: .nombreDeLaClase {propiedad: valor;}

SELECTOR DE ID

selecciona un elemento basándose en un valor de su atributo ID.

sólo puede haber un elemento con un determinado ID dentro de un documento.

no se puede repetir.

ejemplo: #nombreDeID {propiedad: valor;}

SELECTORES CSS BÁSICOS:

  • SELECTOR UNIVERSAL
    • selecciona todos los elementos.
    • opcionalmente puede estar restringido a un espacio de nombre espefífico o a todos los espacios de nombres.
    • Ejemplo: *{propiedad: valor;}
  • SELECTOR DE ATRIBUTO
    • selecciona elementos basándose en el valor de un determinado atributo.
    • Ejemplo: [autoplay]{propiedad: valor;} seleccionará todos los elementos que tengan el atributo autoplay establecido.

SELECTORES SOMBINADOS

  • COMBINADOR DE HERMANOS ADYACENTES
    • el combinador + selecciona hermanos adyacentes.
    • el segundo elementos sigue directamente al primero y ambos comparten el mismo elemento padre.
    • Sintaxis: A + B
    • Ejemplo: La regla h2 + p se aplicará a todos los elementos (p) que siguen directamente a un elemento(h2)
  • COMBINADOR GENERAL DE HERMANOS
    • el combinador - selecciona hermanos.
    • el segundo elemento sigue al primero, no necesariamente de forma inmediata, y ambos comparten el mismo elemento padre.
    • Sintaxis: A - B
    • Ejemplo: La regla p - span se aplicará a todos los elementos (span) que siguen un elemento (p).
  • COMBINADOR DE HIJO
    • el combinador > selecciona los elementos que son hijos directos del primer elemento.
    • Sintaxis: A > B
    • Ejemplo: La regla ul > li se aplicará a todos los elementos (li) que son hijos directos de un elemento(ul).
  • COMBINADOR DE DESCENDIENTES
    • el combinador (espacio) selecciona los elementos que son descendientes del primer elemento.
    • Sintaxis: A B
    • Ejemplo: la regla div span se aplicará a todos los elementos (span) que están dentro de un elemento(div).
  • COMBINADOR DE COLUMNA
    • el combinador || selecciona los elementos especificados pertenecientes a una columna.
    • Sintaxis: A || B
    • Ejemplo: col || td seleccionará todos los elementos (td) que pertenezcan al ámbito de (col).

CASCADA Y HERENCIA CSS

CASCADA

las fuentes de código CSS pueden ser:

  • la hoja de estilos creada por el desarrollador de la página.
  • los estilos predeterminados del navegador.
  • los estilos especificados por el usuario.

HERENCIA

  • los elementos hijos adoptan los estilos determinados para los elementos padre en caso que no se determine uno específico para el elemento hijo.
  • ejemplo: diseño en (body) afectará a todos los elementos dentro, incluyendo párrafos, subtítulos o cualquier otro elemento dentro de (body).

RESETEO DE CSS

la visualización de los elementos estará condicionada por el navegador, motivo por el cual se utilizan herramientas como NORMALIZE O RESET para ajustarlo.

RESETEO

es una solución más agresiva a las diferencias entre los navegadores.

restablece todos los estilos que vienen con el agente de usuario del navegador.

NORMALIZE

es una solución suave.

normalize.css es un pequeño archivo que proporciona coherencia entre navegadores en el estilo de los elementos HTML.

si hay una inconsistencia en uno de los navegadores, los estilos normalize.css arreglarán el estilo del navegador.

DIMENSIONES EN CSS

min o max-height

definimos un mínimo o máximo de ancho de un elemento

LINE-HEIGHT

content-area: el área mínima que ocupa una letra con un aire superior e inferior.

line-height: el área total que ocupa entre líneas.

La propiedad CSS line-height establece la altura de una casilla remarcada por líneas.

Comúnmente se usa para establecer la distancia entre líneas de texto.

Define la altura mínima de las casillas encuadradas por líneas dentro del elemento.

En elementos de línea no reemplazables, especifica la altura que se usa para calcular la altura de la casilla encuadrada por líneas.

UNIDADES DE LONGITUD RELATIVA

Son relativas a algo más.

Se puede lograr que el tamaño del texto se escalen en relación con todo lo demás.

em: Tamaño de letra del elemento padre.

ex: Altura x de la fuente del elemento.

ch: La medida de avance (ancho) del glifo 0 de la letra del elemento.

lh: Altura de la línea del elemento.

vmax: 1% de la dimensión más grande de la ventana gráfica.

Los más utilizados son: em, rem, vh, vw y los porcentajes.

PROPIEDAD VISIBILITY

Es similar a la propiedad display pero mucho más limitada.

Display permite cambiar el tipo de caja de un elemento.

Visibility sólo permite hacer visible o invisible una caja.

VALORES PARA VISIBILITY:

  • VISIBLE: indica que elemento es mostrado
  • HIDDEN: Establece que elemento se esconde, pero los demás elementos de la página se comportan como si el elemento estuviese presente.
  • COLLAPSE: pensada para controlar la visibilidad de elementos de tabla, filas y columnas.

WORD AND LETTER SPACING

WORD-SPACING

Debe estar en formato longitud: cm, px, pt, em, %, normal (por defecto).

No se permiten valores negativos.

Un valor 0 evitará justificación.

WORD-WRAP

Propiedad para controlar las palabras largas que exceden el contenedor, para continuar en la siguiente línea.

VALORES:

  • BREAK-WORD: genera un corte y continúa en la siguiente línea.

OVERFLOW

Propiedad para el control de contenido excedente de elementos y contenedores.

  • VISIBLE: Valor por defecto, siendo todo visible aunque sea desbordado.
  • HIDDEN: El contenido excedente se oculta.
  • SCROLL: El contenido no se desborda y se controla con barras de desplazamiento vertical y horizontal.
  • AUTO: Aparecen automaticamente barras de desplazamiento de ser necesario.
  • INHERIT: Hereda el estilo del elemento padre.

  • AV1: CODEC: Basado en VP10, Daala y Thor. Compite con HEVC/H.265 y soporte bajo.

  • OGV: CODEC: Theora. Alternativa libre a MP4. Con precaución.
  • MKV: CODEC: Matroska. Buena comresión. Potente. No recomendado alto consumo CPU.
  • AVI: CODEC: XviD, DivX 3/5. Menor compresión que MP4. No recomendado, anticuado.

ELEMENTOS DE LÍNEA Y BLOQUE

LA PROPIEDAD DISPLAY

BLOCK VS INLINE

  • Propiedad: Valor
  • display: block; Block elements stack (in column), regardless of their width.
  • display: inline; Inline (filas) elements flow from one line to the next.
  • Propiedad Display: Permite elegir como se muestran los elementos. Como elementos de Línea o de Bloque.
  • Elementos de línea: Todos lo elementos se ubican en una misma línea sin hacer saltos de línea. Ej: enlaces, imágenes, elementos span, textos sub o super, etc.
  • Elementos de bloque: Ocupan el ancho máximo disponible. Realizan salto de línea. Párrafos, bloques div, tablas, formularios, listas, etc.

VALORES PARA DISPLAY CSS

  • INLINE: establece que el elemento debe comportarse como un elemento en línea
  • BLOCK: establece que el elemento debe comportarse como un elemento de bloque.
  • INLINE-BLOCK: El elemento genera una caja de elemento de bloque que fluye con el contenido circundante como si fuera una sola caja en línea
  • NONE: Indica que el elemento no debe ser mostrado.
  • INHERIT Hereda del elemento padre.
  • INITIAL: Indica que se cumpla la forma inicial o predeterminada.
  • ELEMENTOS EN LINEA (INLINE): Elementos cuyo contenido se encuentra en una misma línea. Si no expresamos un salto de línea, los elementos se presentarán uno seguido del otro.
  • ELEMENTOS DE BLOQUE (BLOCK): Se comportan como bloques. Tras cada elemento, se realiza un salto de línea aún sin su especificación.

PROPIEDADES DE POSICIONAMIENTO (CSS POSITION PROPERTY)

Permite posicionar un elemento con respecto a los demás.

  • STATIC: Valor por defecto.
  • RELATIVE Podemos mover al elemento pero tomará como referencia su posición por defecto. Podemos usar la propiedad top, left, right y botompara colocarlo donde queramos.
  • ABSOLUTE: Podemos romper el flujo del documento, definiendo top, bottom, right y left. La diferencia con relative es que empieza desde el inicio del sitio web o del elemento que tenga una posición diferente.
  • FIXED: Quedan fuera del flujo normal de la página. Converte al elemento en flotante, no aceptando ningua de las propiedades left, top, bottom o right. Al desplazarnos el elemento no desaparece. Se utiliza para los menús fijos.
  • STICKY: El elemento es posicionado de acuerdo al flujo normal del documento y luego es desplazado con relación a su ancestro que se desplace más cercano y su bloque contenedor.

VALORES PARA LA PROPIEDAD POSITION:

  • TOP: establece la distancia superior del elemento respecto a otro.
  • LEFT: establece la distancia por el lado derecho del elemento respecto a otro.
  • BOTTOM: establece la distancia inferior del elemento respecto a otro.
  • RIGHT: establece la distancia por lado izquierdo del elemento respecto a otro.

FLEXBOX

  • Es un sistema de elementos flexibles
  • Los elementos de HTML se adaptan y colocan automáticamente, siendo más fácil personalizar los diseños.
  • Especialmente diseñado para crear con CSS estructuras de una sóla dimensión.

Elementos básicos:

  • Contenedor: Es el elemento padre que tendrá en su interior cada uno de los ítmes flexibles. Establecemos las propiedades al elemento padre.
  • Eje principal: Los contenedores tendrán una orientación por defecto horizontal (en fila).
  • Eje secundario: Los contenedores tendrán una orientación secundaria, perpendicular a la principal. Si la principal es horizontal, la secundaria será vertical.
  • Ítem: Cada uno de los hijos que tendrá el contenedor en su interior.

PROPIEDAD DISPLAY

  • Define un contenedor flexible en línea o bloque según el valor.
  • Permite un contexto flexible para todos sus hijos.
  • Posibles valores:
    • Flex
    • Block
    • Inline
    • Inline-Block
    • Table
    • Grid
  • Para Flexbox siempre aplicaremos al contenedor la propiedad display: flex;

DIRECCIÓN DE LOS EJES

Existen 2 propiedades principales para manipular la dirección y comportamiento de los ítems a lo largo del eje principal del contenedor.

  • PROPIEDAD: Valor. Significado
  • FLEX-DIRECION: Row, Row-reverse, Column, Column-reverse. Cambia la orientación del eje principal.
  • Podemos modificar la dirección del eje principal del contenedor para que se oriente en horizontal o vertical. sufijo reverse indica que coloque los ítems en orden inverso.
  • FLEX-WRAP Nowrap, Wrap, Wrap-reverse. Evita o permite el desbordamiento.

FLEX-DIRECION

Permite tener control muy alto sobre el orden de los elementos en una página.

  • ROW: Establece la dirección del eje principal en horizontal.
  • ROW-REVERSE: Establece la dirección del eje principal en horizontal invertido.
  • COLUMN: Establece la dirección del eje principal en vertical.
  • COLUMN-REVERSE: Establece la dirección del eje principal en vertical invertido.

FLEX-WRAP

Para especificar el comportamiento del contenedor respecto a evitar que se desborde o permitir que lo haga.

  • NOWRAP: Establece los ítems en una sola línea no permitiendo que se desborde el contenedor.
  • WRAP: Establece los ítems en modo multilínea permitiendo el desborde del contenedor.
  • WRAP-REVERSE: Establece los ítems en modo multilínea, pero en dirección inversa.

FLEX-FLOW

  • Es una propiedad de atajo (short-hand)
  • Para resumir los valores de las propiedades flex-direction y flex-wrap.
  • Se especifica en una sola propiedad, ahorrándonos utilizar las propiedades.
  • FLEX-FLOW: row, row-reverse, column, column-reverse
  • FLEX-FLOW: nowrap, wrap, wrap-reverse

    GRIDS

    • Es un sistema de maquetación web que divide la página en una cuadrícula o rejilla, grid, para posicionar los elementos.
    • Por su practicidad es un estandar.
    • Casi cualquier navegador soporta este código.

    ELEMENTOS

    • CONTENEDOR: El elemento padre contenedor que definirá la cuadrícula.
    • ÍTEM: Cada uno de los hijos que contiene la cuadrícula.
    • CELDA (GRID CELL): Cada uno de los cuadritos, unidad mínima de la cuadrícula.
    • ÁREA (GRID AREA): Conjunto de celdas de la cuadrícula.
    • BANDA (GRID TRACK): Banda horizontal o vertical de celdas de la cuadrícula.
    • LÍNEA (GRID LINE): Separador horizontal o vertical de las celdas de la cuadrícula.

    (div class="grid") (contenedor)

    (div class="a")Item 1(/div) (cada uno de los ítems del grid)

    (div class="b")Item 2(/div)

    (div class="c")Item 3(/div)

    (div class="d")Item 4(/div)

    (/div)

    VENTAJAS DE CSS GRID:

    • Controla los elementos en las 2 diminsiones.
    • Menos código, menos bugs, revisiones más sencillas y menor probabilidad de bugs.
    • Optimiza recursos: Código más simple y consistente.
    • Responsive más sencillo. Se puede crear elementos dinámicos que se adaptan a diferentes tamaños sin complicaciones.

    COMO APLICAR CSS GRID

    • Hay que utilizar sobre el elemento contenendor la propiedad display y especificar el valor grid o inline-grid.
    • TIPO DE ELEMENTO: Descripción
    • INLINE-GRID: Establece una cuadrícula con ítems en línea, de forma equivalente a inline-block.
    • GRID: Establece una cuadrícula con ítems en bloque, de forma equivalente a block.

    GRID CON FILAS Y COLUMNAS

    • Es posible crear cuadrículas con un tamaño explícito.
    • PROPIEDADES: Valor. Descripción
    • GRID-TEMPLATE-COLUMNS: [col1] [col2]... Establece el SIZE de cada columna
    • GRID-TEMPLATE-ROWS: [fila1] [fila2]... Establece el SIZE de cada fila

    Grid con filas y columnas:

    .grid {

    display: grid;

    grid-template-columns: 50px 300px 1fr;

    grid-template-rows: 200px 2fr 5% 1fr;

    }

    • Con estas propiedades definimos el número de columnas (grid-template-columns) y fílas (grid-template-rows)
    • En el ejemplo tenemos 3 columnas y 4 filas.
    • La unidad fraccionaria (fr) es una unidad relativa que nos permite definir un cierto porcentaje de la totalidad del contenendor.
    • Si tengo 2 columnas (1fr, 3fr) se debe sumar las dimensiones que será la totalidad del contenedor

    ESPACIOS EN GRIDS

    • ROW-GAP: Podemos definir espacios entre filas. Solo debemos indicar un valor numérico. Ej:rows-gap:10px;
    • COLUMNS-GAP: Con esta propiedad podemos definir espacios entre columnas, sólo debemos indicar un valor numérico. Ej: columns-gap:10px;
    • GAP Definir espacios entre filas y columnas en unidades relativas (%, em, rem, etc) y absolutas(px,etc)
    • Hay que definir un valor para los espacios de filas y otro para columnas.
    • Ej: gap:10px 20px;

    Define una plantilla de cuadrícula haciendo referencia a los nombres de las áreas de cuadrícula que se especifican con la propiedad grid-area.
