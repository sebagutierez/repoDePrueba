2. BASES

2.1 Se puede escribir las etiquetas en mayúsculas, pero no es una buena práctica.

2.2 Las etiquetas en celeste son propiedades o atributos.

2.3 Las etiquetas en marrón son los valores de las propiedades o atributos.

2.4 Sólo se debe escribir dentro de las etiquetas head o body.

2.5 La propiedad (html lang="es") especifica que el idioma es español o en inglés lang="en".

2.6 En head se detalla la información del sitio. Etiquetas meta, title, script, etc. Estas etiquetas no serán visualizadas en el sitio.

2.7 Dentro de la etiqueta body se incluye lo que será visualizado en la web.

2.8 (meta charset="UTF-8"): Charset= Tipo de codificación. UTF-8 Permite la utilización de caracteres españoles.

2.9 (meta name="viewport" content="width=device-width, initial-scale=1.0") Especifica que el ancho se adapte a los dispositivos que visualizarán el contenido.

2.10 La etiqueta HTML contiene las etiquetas HEAD y BODY.

2.11 Las carpetas y los archivos no deberían contener ni mayúsculas ni espacios, utilizando guión bajo o medio, para evitar incompatibilidades o errores.

2.12 index.html Siempre debe ser hijo directo y las demás páginas y archivos dentro una carpeta.

2.13 No es una buena práctica generar varios niveles en las carpetas, para que la arquitectura no sea compleja ya que el usuario debe hacer muchos clics. Hasta 3 clics hace un usuario para que se vaya. Por eso existen web que tiene todo el contenido en la landing page, para que el usuario no tenga que hacer muchos clics.

2.14 Es una buena práctica crear un sólo h1

Su estructura es de TIPO ARBOL y cada uno de los elementos representa un nodo.
