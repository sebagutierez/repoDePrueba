ENTIDADES HTML

  • Las entidades son estructuras que proporcionan un forma de representar símbolos.
  • Hay dos maneras de codificar una entidad HTML.
    1. &nombre_entidad;
    2. &#número_entidad;

Esta bicicleta tiene un costo de &dolar;40

Esta bicicleta tiene un costo de $40

El perímetro de un círculo es 2Π

El perímetro de un círculo es 2n

Nuestro corres electrónico:test@test.com

Este es mi correo electrónico: test@test.com

ETIQUETAS DE CÓDIGO

  • (code): se emplea para representar un código fuente usando un tipo y tamaño de letra para proporcionar un formato adecuado a una pieza de programa de computación.

ETIQUETA IFRAME

  • Representa un contexto de navegación anidado.
  • El comportamiento de iframe es parecido a la etiqueta image y admiten el atributo align para su posicionamiento o pueden ser posicionadas mediante CSS.
  • Atributos:
    • height="altura" Altura en pixeles de la ventana donde se muestra el iframe.
    • width="anchura" Anchura en pixeles o en %. De no especificarlo, debenderá del navegador.
    • scrolling="yes" "no" "auto" Barras de desplazamiento. El valor por defecto es auto.
    • name="nombre" Da nombre a la etiqueta, lo que será imprescindible si queremos direccionar los enlaces hacia la ventana del iframe.

ETIQUETAS SCRIPT Y LINK

Etiquetas para integrar CSS y JavaScript en el código.

PARA ENLAZAR CSS

  • (style): para insertar código CSS dentro de (head)
  • (link): para enlazar un archivo con un CSS externo dentro de (head)
  • (link rel="stylesheet" type="text/css" href="estilo.css">
  • Enlaza el archivo CSS estilo.css

PARA ENLAZAR JAVASCRIPT

  • (script): para insertar código escrito en JavaScript
  • (script con atributo src): para enlazar un archivo con JavaScript externo
    • (script src="funciones.js")(/script)
    • Enlaza el archivo JavaScript funciones.js
  • JavaScript en etiquetas:Podemos asociar código JavaScript que responda a ciertos eventos.
    • Por ejemplo se emplea cuando deseamos ejecutar un código JavaScript al hacer click en un enlace.
      • Presiona
    • (a onclick="alert(1)")Presiona(/a)
    • Al darle click ejecuta alert(1) que despliega una ventana con el contenido 1.

ETIQUETAS DE VIDEO

  • Source: (scr) dirección_url del video a reproducir.
  • poster="dirección_url" Muestra una imágen a modo de presentación.
  • preload="auto" "metadata" "none" Indica como realizar la precarga del video.
  • mediagroup="nombre" Establece un nombre para un grupo de contenidos multimedia.
  • autoplay="boolean" Comienza a reproducir el video automaticamente.
  • muted="boolean" Muestra los controles de reproducción.
  • controls="boolean" Muestra los controles de reproducción.
  • width="tamaño" Indica el tamaño de ancho del video.
  • hight="tamaño" Indica el tamaño de alto del video.
  • Ejemplo: (video src="numen.mp4" width="1920" hight="1080")(/video)
  • Mostrará el primer fotograma del video con el tamaño especificado, pero se verá como una imagen porque no muestra los controles del video y tampoco tiene la reproducción automática activada. Se deberá aguregar los atributos controls o autoplay.

FORMATOS DE VIDEO

  • Un archivo de video tiene 2 partes. El formato contenedor, el formato de video, y en su interior puede tener múltiples componentes codificados con diferentes codecs.
  • Un video básico suele tener como mínimo un componente de video y otro de audio, pero puede tener muchos más.
    • subtítulos
    • imágenes, etc.
  • Dependiendo del formato de un video, puede que sea factible utilizarlo para web o no.
  • (source): Puede tener los atributos src, type y establece un archivo de video o lo añade como alternativa.
  • (track) tiene los atributos src, srclang, label, king, default y establece un archivo de subtítulos o lo añade como alternativa.
  • (video) La etiqueta como contenedora nos permite incluir etiquetas (source) en su interior para proporcionar formatos alternativos y tener mayor compatibilidad con otros navegadores
    • (video width="640" height="480")
    • (source src="video.mp4" type="video/mp4" /)
    • (source src="video.webm" type="video/webm" /)
    • (source src="image.png" alt="Video no soportado"/) Su navegador no soporta contenido multimedia(/video)
  • Los navegadores no mostrarán todos los contenidos a la vez, sino que seguirán el siguente procedimiento:
    1. intenta mostrar el primer formato MP4. Si el navegador no lo soporta, salta al siguiente.
    2. intenta mostrar el 2º formato WEBM. Si el naveador no lo soporta, salta al siguiente.
    3. intenta mostrar el 3º formato OGV. Si el navegador no lo soporta, salta al siguiente.
    4. si se trata de un navegador que no soporta HTML5, intentará mostrar la imágen.
    5. si se trata de un navegador de terminal de texto o sin capacidades gráficas, mostrará el texto: "Su navegador no soporta contenido multimedia"
    6. De esta manera tendremos soporte completo para todo tipo de dispositivos.

ETIQUETA AUDIO

  • Source: src="dirección_url" Audio que va a reproducirse.
  • preload="auto" "metadata" "none" Indica como realizar la precarga del audio.
  • mediagroup="nombre" Establece el nombre para un grupo de contenidos multimedia.
  • autoplay="boolean" Comienza a reproducir el audio automáticamente.
  • muted="boolean" Muestra los controles de reproducción.
  • controls="boolean" Muestra los controles de reproducción.
  • loop="boolean" Vuelve a iniciar el audio cuando finaliza su reproducción.
  • (audio src="audio.mp3")(/audio)
  • (audio src="audio.mp3" preload="none" controls)(/audio)
    • cargamos un audio mp3, pero indicamos que no precargue nada.
    • empezará a descargarse sólo cuando el usuario pulse en los controles de reproducción.
    • para evitar consumo de ancho de banda de archivos que es probable que el usuario no escuche o en dispositivos móviles donde las tarifas de datos tiene costo.
  • (audio src="audio.ogg" autoplay loop)(/audio)
    • carga un archivo de audio OGG y lo reproduce automáticamente y en bucle, de modo que vuelve a empezar cuando termina.

FORMATOS DE AUDIO

  • Listado de formatos o codecs de audio más utilizados:
    • MP3 (MPEG Layer-3): tiene buena calidad y soporte.
    • ACC (Advanced Audio Coding): mejora el MP3. usado como audio en MP4, tiene buen soporte.
    • OGG (Ogg Vorbis): Buena calidad, alternativa a MP3. soporte medio.
    • Opus (Opus): Buena calidad, alternativa a MP3. soporte medio.
    • FLAC (FLAC Audio Lossless): Compresión sin pérdidas. Alto tamaño. Buen suporte.
    • WAV (Wave sound): Formato de Microsoft. Está soportado pero es muy pesado.

AUDIO (AVANZADO)

  • Audio puede ser utilizada como etiqueta contenedora e incluir varias etiquetas HTML para tener capacidades adicionales.
  • (source): tiene los atributos src y type. Establece un audio o lo añade como alternativa.
  • (track): Tiene los atributos src, srclang, label, kind, default. Establece un archivo de subtítulos o lo añade como alternativa.
  • Pasamos de un esquema donde sólo usabamos una etiqueta (audio) a otro un poco más avanzado, donde (audio) contiene multiples etiquetas que actúan sobre ese audio en cuestión.

AUDIO ALTERNATIVO (FALLBACK)

si utilizamos la etiqueta (audio) como etiqueta contenedora, podemos incluir etiquetas (source) en su interior para proporcionar formatos alternativos y tener mayor compatibilidad con otros navegadores.

(audio)

(source src="audio.opus"/)

(source src="audio.ogg"/)

(source src="audio.mp3"/)

(/audio)

En este ejemplo el navegador intentará reproducir el archivo de audio de formato OPUS.

En caso de no estar soportado por el navegador intentará reproducir el formato Ogg vorbis

En caso de tampoco soportarlo, reproducirá el formato MP3.

QUE ES UN SVG

  • Gráficos Vectoriales Redimensionables (Scalable Vector Graphic)
  • Son una especificación recomendada por el Word Web Consortum, W3C, desde 2001 como estándar para describir gráficas de 2 dimensiones en formato XML.
  • Generar un formato de archivo para describir elementos gráficos vectoriales.
  • Varias empresas como Microsoft con el VML, Vector Markup Language y Adobe en coordinación con IBM, Netscape y SUN con su PGML (Precision Graphic Markup Language), participaron en las versiones previas de lo que se convirtió en SVG.
  • VENTAJAS
    • generar archivos que ocupen poco espacio en la memoria y puedan comprimirse de manera óptima.
    • permitir que las imágenes puedan escalarse a cualquier tamaño sin perder nitidez.
    • facilitar la creación de gráficas tanto estáticas como animadas.
    • pueden ser del siguiente tipo:
      • figuras vectoriales
        • trazos que consisten en líneas rectas y curvas
      • imágenes
      • textos
    • las imágenes SVG pueden agruparse, estilizarse, transformarse y componerse de objetos renderizados.
    • Dentro de sus características principales se encuentran las transformaciones anidadas, trazos recortados, capas de transparencia, efectos de filtro y objetos de pantilla.
    • los dibujos SVG pueden ser interactivos y dinámicos,
    • sus animaciones se declaran o utilizando lenguages script como JavaScript.

CANVAS

  • (canvas) es una etiqueta que permite definir un lienzo para dibujar imágenes usando algún lenguaje de script.
  • en general se utiliza javascript.
  • instrucciones para dibujar:
    • Figuras (círculos, rectángulos, etc)
    • Texto
    • Imágenes
    • Colores de fondo, etc.
  • Canvas es una alternativa a SVG y generalmente se utiliza para generar gráficos dinámicos en el momento.

Canvas también es usado ara dibujar en 3D con la tecnología WebGL
